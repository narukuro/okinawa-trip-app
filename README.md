# あかブロック脱出

赤い大ブロック（2×2）を盤の下中央の出口へ運んで脱出させる、箱入り娘（Klotski）型のスライドパズル。インストール不要で遊べる Web アプリ（PWA）です。

▶ **プレイ：** https://narukuro.github.io/okinawa-trip-app/

---

## 概要
- ジャンル：箱入り娘（Klotski）型スライドパズル
- 目的：**赤い大ブロック（2×2）を盤の下中央の出口へ運べばクリア**
- 形態：インストール不要の Web アプリ（PWA）。iPhone の Safari／ホーム画面アプリで動作、オフライン対応
- 配信：GitHub Pages。`main` へ push すると GitHub Actions で自動デプロイ

## 技術構成
- バニラ構成（フレームワーク無し）：HTML / CSS / JavaScript
- `index.html`：画面構造、`style.css`：デザイン、`app.js`：ゲーム本体
- `solver.js`：パズル生成＆最短手数ソルバー（Web Worker 兼用）
- `sw.js`：Service Worker（オフライン用キャッシュ＋更新制御）
- `manifest.json`：PWA 設定、アイコン各種

## 盤面・ルール
- 盤：**4 列 × 5 行**
- ブロックの種類と色：
  - 赤（コーラル）：主役 2×2（破線リング付きで明示）
  - 青：縦長 1×2 ／ 緑：横長 2×1 ／ 黄（砂色）：1×1
- 出口：下辺の中央 2 マス（盤幅の 50%）
- 操作：指でスライド（空いている方向へドラッグ）。盤の外には出せない
- クリア：赤が出口位置（下中央）に収まった瞬間＝脱出成功
- 手数の数え方：**1 回のスライド＝1 手**（一方向に何マス動かしても 1 手）

## レベル制と難易度
- レベル自動生成・実質無限。クリアごとにレベルアップ
- 各レベルに「目標手数」を表示（「レベル N・目標◯手で脱出！」）
- 難易度カーブ：レベルとともに必要手数が上昇 → **3, 4, 5, …, 13** と上がり、以降 13 で据え置き
  - 13 は 4×5 盤で「ぴったりの手数」を安定生成できる上限

## 死守する 2 大原則（仕様として保証）
1. **レベルが上がると難しくなる（手数は絶対に下がらない）**
   - 前レベルの手数を下限（floor）として生成に渡し、floor 以上の盤しか採用しない
   - 手数に上限（13）を設け、跳ね上がり → 翌レベルで下がる現象を防止
   - アプリ側も、下限未満のキャッシュ／生成結果は破棄して再生成
2. **表示手数で本当に解ける**
   - 「目標◯手」は生成後に BFS で実測した最短手数そのもの。必ずその手数で解け、それ未満では解けない

検証：6 通し × 30 レベルの連続生成で **減少 = 0／表示手数の不一致 = 0／盤の重なり = 0**。

## パズル生成ロジック（solver.js）
- 逆再生方式：解けた状態（赤が出口）からランダムに手を戻して問題化 → 必ず解けることを保証
- 盤の混み具合で難易度を調整（手数が多いほどブロックを詰める）
- BFS（幅優先探索）で各盤面の最短手数を実測し、目標手数に合うものだけ採用
- Web Worker で生成＋次レベルを裏で先読みして切替を一瞬に。生成は概ね 1 秒前後、失敗時は自動リトライ
- 同種ブロックを同一視する盤面ハッシュで探索を効率化

## UI・演出
- デザイン：生成り（オフホワイト）× コーラルのミニマル＆温かみのある配色、丸みのあるブロック（Zen Maru Gothic / M PLUS Rounded 1c）
- アニメーション：移動時のバウンド、手詰まり時のブルブル、クリア時の「赤が出口へスーッと退出」
- クリア画面：★評価（最少手数 = ★3）＋クリア手数／最少手数の表示＋「次のレベル」
- ステータス表示：レベル／手数／ベスト（最高到達レベル）
- 各種 iPhone サイズで画面内に収まるよう自動フィット

## データ保存（端末ローカルのみ／サーバー無し）
- `localStorage`
  - `akablock_best_level`：ベスト（最高到達レベル）
  - `akablock_debug`：デバッグモード ON/OFF
- `Cache Storage`：`akablock-levels-v11`（オフライン用のファイル一式）
- プレイ途中の状態は保存しない（リロードでレベル 1 から）。端末・ブラウザごとに独立、同期なし

## PWA・配信・更新
- オフライン対応（Service Worker がファイルをキャッシュ）
- 取得戦略は「ネットワーク優先・オフライン時のみキャッシュ」→ オンラインなら常に最新コードに更新
- 更新時はキャッシュ名をバージョンアップ（現在 v11）して確実に差し替え

## iPhone での遊び方（ホーム画面に追加）
1. このサイトを iPhone の **Safari** で開く
2. 共有ボタン → **「ホーム画面に追加」**
3. 追加されたアイコンから起動すると全画面のアプリとして遊べます

## デバッグモード
- 起動：URL に `?debug=1`、またはタイトルを 5 回タップ（一度 ON にすると記憶、× で解除）
- 機能：
  - レベルジャンプ（任意レベルへ移動）
  - ヒント（次の一手）：最善手のブロックを光らせ、方向と残り最短手数を表示
  - 自動で解く（デモ）：最短手順をアニメ再生して脱出まで実演

## ファイル構成
```
index.html      画面構造
style.css       デザイン・アニメーション
app.js          ゲーム本体（操作・描画・レベル進行・デバッグ）
solver.js       パズル生成＆最短手数ソルバー（Web Worker 兼用）
sw.js           Service Worker（オフライン・更新制御）
manifest.json   PWA 設定
icon-192.png / icon-512.png / icon-180.png   アイコン
.github/workflows/deploy.yml   GitHub Pages 自動デプロイ
```

## ローカルでの動かし方
Service Worker と Web Worker を使うため、`file://` ではなく簡易サーバー経由で開きます。

```sh
# リポジトリのルートで
python3 -m http.server 8123
# ブラウザで http://localhost:8123/ を開く
# デバッグモード: http://localhost:8123/?debug=1
```

## 今後の拡張余地
- 手数の上限は 13（4×5 盤の安定生成上限）。さらに上げ続けるなら盤を 5×6 へ拡張（手数 20 超も可能・ブロックは小さくなる）
- 続きから再開（途中状態の保存）、タイム計測／レベル別ベスト手数、効果音・クリアの紙吹雪 など
